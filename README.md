# 物料仓库管理

一个用于管理个人物料 / 元器件库存的桌面应用，支持本地存储、Tauri 本地备份、Supabase 云同步和 OBS 对象存储备份。

## 功能特性

- 📦 物料管理：添加、编辑、删除物料条目
- 🔎 关键词搜索：名称、型号、分类、项目等字段搜索
- 🧭 分类与位置：支持自定义分类、仓位、项目
- 📊 库存管理：数量增减、低库存预警
- 📎 资料链接：支持供应商链接、规格书链接、图片链接
- ☁️ 云同步：支持 Supabase 云端表同步
- 🧾 备份导出：自动生成本地 `warehouse-backup.json`、`warehouse-backup.xls`
- 🌐 OBS 备份：支持对象存储（OBS）直接上传/下载快照与图片
- 💻 桌面应用：基于 Tauri 的原生桌面体验

## 技术栈

- React + TypeScript + Vite
- Tauri 桌面应用
- Supabase 云数据库
- OBS 对象存储（支持 JSON 快照与物料图片）
- @dnd-kit 拖拽排序支持

## 环境配置

### Supabase 云同步（可选）

应用支持将数据同步到 Supabase。如果未配置，数据默认保存在浏览器本地存储或 Tauri 本地存储。

1. 注册并创建 Supabase 项目
2. 获取项目 `URL` 和 `anon key`
3. 启用 Email 登录认证
4. 在 Supabase SQL 编辑器中创建物料表：

```sql
create table materials (
  id bigint generated by default as identity primary key,
  name text not null,
  model text,
  category text not null,
  description text,
  quantity integer default 0,
  location text,
  created_at timestamp with time zone default timezone('utc'::text, now()) not null,
  updated_at timestamp with time zone default timezone('utc'::text, now()) not null
);

alter publication supabase_realtime add table materials;
```

如果需要用户备份与恢复功能，可选创建备份表：

```sql
create table if not exists warehouse_backups (
  id uuid primary key references auth.users(id) on delete cascade,
  user_id uuid not null references auth.users(id) on delete cascade,
  snapshot jsonb not null,
  updated_at timestamptz not null
);

alter table warehouse_backups enable row level security;

create policy "Users can read own backup"
on warehouse_backups for select
using (auth.uid() = user_id);

create policy "Users can create own backup"
on warehouse_backups for insert
with check (auth.uid() = user_id and auth.uid() = id);

create policy "Users can update own backup"
on warehouse_backups for update
using (auth.uid() = user_id)
with check (auth.uid() = user_id and auth.uid() = id);
```

5. 配置环境变量或在应用内填写 Supabase：

```env
VITE_SUPABASE_URL=https://你的项目.supabase.co
VITE_SUPABASE_ANON_KEY=你的 anon public key
```

### OBS 对象存储备份（可选）

应用支持将备份快照和物料图片上传到 OBS。

需要配置以下参数：

- OBS Endpoint
- OBS Bucket
- OBS Object Key（默认 `warehouse/warehouse-backup.json`）
- Access Key ID
- Secret Access Key

OBS 备份会将当前快照写入对象存储，并可下载恢复。

### 本地备份目录

桌面版应用会在系统应用数据目录下创建 `backups/warehouse-backup.json` 和 `backups/warehouse-backup.xls`。

可以通过应用设置指定自定义备份目录。

## 主要使用说明

### 添加 / 编辑物料

- 点击“添加物料”按钮
- 输入名称、型号、分类、封装、参数、供应商、价格、链接、项目、库存、位置等字段
- 保存后会自动写入本地存储
- 点击已有物料条目即可编辑或删除

### 搜索与筛选

- 使用顶部搜索框输入关键词
- 支持按名称、型号、分类、项目等内容搜索
- 可使用分类与位置筛选当前物料列表

### 库存管理

- 通过 +1 / -1 操作调整数量
- 支持设置低库存阈值，便于发现缺货物料
- 所有变更会记录到操作日志中

### 图片与资料链接

- 可以为物料添加图片 URL
- 也可以录入供应商链接、规格书链接，方便查阅
- OBS 配置后，可将物料图片复制到 OBS 存储并生成内网访问 URL

### 日志与快照

- 应用会记录物料变更操作日志
- 保存快照时会在本地 `localStorage` 中写入完整数据
- 如数据有变更，应用会自动保留上一版本快照，便于恢复

### 云端同步与恢复

- 填写 Supabase `URL` / `anon key` 并登录后启用云同步
- 支持上传当前快照到 Supabase
- 在新设备或重新安装后，可通过“从云端恢复”恢复数据
- 若未登录或未配置 Supabase，则数据仍保存在本地

### 备份导出

- 桌面版会自动生成 JSON 与 Excel 备份文件
- 本地备份文件位于应用数据目录或自定义目录
- `warehouse-backup.xls` 可直接用 Excel 打开查看备份数据

## 开发者说明

### 脚本

- `npm run dev`：启动 Vite 开发服务器
- `npm run build`：构建前端生产文件（包含 TypeScript 编译）
- `npm run preview`：预览构建结果
- `npm run tauri dev`：以 Tauri 桌面应用模式运行
- `npm run tauri build`：构建 Tauri 桌面发行包

### 依赖说明

- `@supabase/supabase-js`：Supabase 云数据库客户端
- `@tauri-apps/api`：桌面文件、对话框、路径操作
- `@tauri-apps/cli`：Tauri 构建与调试工具
- `@dnd-kit/core` / `@dnd-kit/sortable`：支持拖拽排序交互

## 注意事项

- 未配置 Supabase 时，应用默认使用浏览器本地存储
- 桌面版路径权限受 Tauri allowlist 控制，仅允许应用数据目录读写
- OBS 需确保 CORS、权限、桶配置正确，否则上传/下载失败
- 若要部署到云端，请确保 Supabase 和 OBS 的 key 不泄露
